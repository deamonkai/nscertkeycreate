cert_store

Configuration for SSL certificate store on NetScaler Console resource.

Read/write properties

created_time<Double>

Creation time for this activity in seconds.

managed_by<String>

Indicates the CA provider if the certificate is managed by a ACME CA provider..

key_file<String>

Key file name.
Minimum length = 1
Maximum length = 128

acme_ca_config_id<String>

ID of the CA config that is used to renew the certificate.

certchain_file_id<String>

Comma separated list of IDs against which cert chain contents are stored in cert_store_data table.

password_sync_required<Boolean>

Set to true, if the password update is required for this certificate. This is applicable when the certificate is discovered from ADC..

mode<Integer>

To identify if the certificate is added manually or discovred from NetScaler (1.Manual/2.Discovered).
Minimum value = 1
Maximum value =

zero_touch_certificate<Boolean>

Set to true, if the certificate is uploaded for zero touch ssl certificate repository purpose..

last_renewed_at<Double>

Last renewal time for this certificate in seconds. This is applicable when the certificate is enabled for auto renewal..

domain<String>

Domain name from the subject of the certificate.

id<String>

Id is system generated key for all certificate story entries. .

is_issued_from_csr<Boolean>

Set to true, if the certificate is issued from CSR..

updated_time<Double>

Recent status updated time for this activity in seconds.

csr_data<String>

Base64 encoded CSR file contents..
Maximum length = 16384

issued_by_org<String>

Issued Organization from the subject of the certificate.

name<String>

Name or alias which identifies certificate and key pair.
Maximum length = 128

renewal_status<String>

Last renewal status (SUCCESS/FAILED) for this certificate. This is applicable when the certificate is enabled for auto renewal..

cert_type<String>

Type/kind of the certificate. 1) server_cert, 2) intermediate_cert, 3) root_cert, 4) key. This is applicable for zero_touch files. For cert_store, it will be always server_cert..

renew_mode<String>

Renew mode for the certificate. It can be MANUAL or AUTO. If it is set to AUTO, then the certificate will be auto renewed based on the CA config associated with this certificate..

dns_provider_name<String>

DNS Provider of the domain name from the issuer certificate..

bound_entities_number<Integer>

Number of entities to which a certificate in store is bound.

cert_signing_type<String>

Certificate signature.

cert_issuer_type<String>

Issuer of the certificate.

cert_id_array<String[]>

Array of cert store IDs.

update_cert_chain<Boolean>

Set to true,if certificate chain needs to be updated during update operation.

cert_info_required<Boolean>

This will return detailed certificate info like issues, algorithm, used/unused, etc.

update_cert_on_devices<Boolean>

The certificate will be applied on all the associated NetScaler(s)when this flag is set to true.

password<String>

The pass-phrase that was used to encrypt the private-key..
Maximum length = 128

cert_signature_algo_type<String>

Certificate algorithm.

cert_keysize_type<String>

Size of the public key.

certchain_data<cert_store_data[]>

An array of Base64 encoded certificate chain contents..

end_renew_time<Double>

End time of certificate renew in seconds.

save_config<Boolean>

true, if save config is required.

update_cert_and_key<Boolean>

Set to true,if certificate and key files needs to be updated during update operation.

acme_ca_config_name<String>

Name of the CA config that is used to renew the certificate.

start_renew_time<Double>

Start time of certificate renew in seconds.

cert_used_type<String>

The cert is used or not used.

is_acme_feature<Boolean>

This is to identify the ACME workflow and returns only the server certificates (skips IC, CA and Keys) from the cert store and zero-touch certificate repository.

no_domain_check<Boolean>

Specify this option to override the check for matching domain names during certificate update operation.

update_password_only<Boolean>

Set to true, if only password needs to be updated during update operation.

certificate_ids<String[]>

Array of certificate IDs.

cert_data<cert_store_data>

Base64 encoded certificate file contents..

key_data<String>

Base64 encoded key file contents..
Maximum length = 16384

Read only properties

store_code<Integer>

0 indicates no error.100: Failure in storing Server certificate file.200: Failure in storing key file. 300:Failure in storing password.400: Failure in storing certificate chain.

version<Integer>

Version.

public_key_algorithm<String>

Public Key Algorithm.

serial_number<String>

Serial Number.

subject<String>

Subject.

issuer<String>

Issuer.

valid_from<String>

Valid From.

subjaltname<String>

Comma separated Subject Alternative Names.

public_key_size<Integer>

Public Key Size.

hash_code<String>

Zero touch: MD5sum(hash code) of the complete certificate (certificates, key and chain certiticates) or key content.

signature_algorithm<String>

Signature Algorithm.

cert_format<String>

Certificate Format.

valid_to<String>

Valid To.

revision_number<Integer>

Certificate revision number.

days_to_expiry<Integer>

Days before SSL certificate expires.

status<String>

Tells whether the certificate is still valid or not.

activity_id<String>

Activity Id used to track the progress of certificate operations..

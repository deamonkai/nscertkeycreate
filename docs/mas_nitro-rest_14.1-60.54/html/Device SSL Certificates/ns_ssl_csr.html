ns_ssl_csr

Configuration for CSR File resource.

Read/write properties

keyfile<String>

Key File.
Minimum length = 1
Maximum length = 128

is_deploy<Boolean>

Whether Auto deployment of certificates managed by third party CA should be done by NetScaler Console..

file_location_path<String>

File Location on Client for download.
Minimum length = 1

keyform<String>

Key Form.

app_name<String>

Application name.Used for tracking task logs created for a specific application.

subjaltname<String>

Subject alternative name.

organizationunit<String>

Organization Unit.

policy<String>

Applicable only if the issuer is Venafi.Policy folder path which determines the certificate attributes..

passphrase<String>

Pass Phrase used to encrypt the private-key.

commonname<String>

Common Name.
Minimum length = 1

keystrength<String>

Key bits in size, if algorithm is RSA.EllipticCurve, if algorithm is ECC.

city<String>

City.

statename<String>

State Name.
Minimum length = 2
Maximum length = 128

file_name<String>

Name for and, optionally, path to the certificate signing request (CSR).
Minimum length = 1
Maximum length = 256

is_acme_feature<Boolean>

True if the key file is used for ACME feature.

certificate_name<String>

Name of the certificate.

countryname<String>

Country Name.
Minimum length = 2
Maximum length = 128

keyalgorithm<String>

Key Algorithm.Possible values: RSA, ECC.

organizationname<String>

Organization Name.
Minimum length = 1

issuer<String>

Third party Certificate Authority provider.

certificate_dn<String>

Applicable only if the issuer is Venafi. Certificate path is populated in this field..

Read only properties

file_size<Integer>

file_size.

tenant_name<String>

Tenant Name.

csr<String>

Generated CSR.

file_last_modified<String>

Last Modified.

activity_id<String>

Activity Id used to track the progress of create certificate.

file_last_modified_epoch<Double>

Last Modified (Epoch).

serial_number<String>

Serial Number.

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE

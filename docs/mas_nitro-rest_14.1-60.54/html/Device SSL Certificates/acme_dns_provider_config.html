acme_dns_provider_config

Configuration for ACME DNS provider configuration resource.

Read/write properties

dns_credentials_map<acme_access_parameters[]>

Array of key and value for DNS credentials. The key is the credential name and the value is the credential value..

id<String>

Id is system generated key for DNS provider configuration..

is_active<Boolean>

Indicates if this DNS Provider configuration is active (configuration is complete). If false, it will not be used for ACME challenges..

dns_max_retries<Integer>

Maximum number of attempts to check for DNS record existence during ACME challenge verification..
Minimum value = 3
Maximum value =

dns_sleep_time<Integer>

One-time sleep duration (in seconds) to wait after creating DNS records before starting ACME challenge verification for all domains..
Minimum value = 20
Maximum value =

is_default<Boolean>

Indicates if this is the default DNS Provider configuration.

acme_dns_name<String>

ACME DNS name. This is the name used by acme.sh client to identify the DNS provider in ACME challenges..
Maximum length = 128

domain_pattern<String>

Domain pattern for which this DNS provider is applicable. Example: *.example.com.

dns_provider_name<String>

DNS Provider name. Values: 1.AWS_Route53, 2. Google_Cloud_DNS, 3. Azure_DNS.
Maximum length = 128

certificate_list<acme_dns_cert_map[]>

Array of certificate IDs associated with this DNS Provider configuration. This is used to link the DNS Provider with the certificates that use it for ACME challenges..

acme_ca_config_id<String>

ID for CA Configuration.

domain_info_required<Boolean>

Set to true, if the DNS provider requires domain information to be provided for each DNS Provider.

certificate_count<Integer>

Number of certificates/domain associated with this DNS Provider configuration.

domain_count_required<Boolean>

Set to true, if the DNS provider requires domain count to be provided for each DNS Provider.

renew_mode<String>

Renew mode for the certificate. It can be MANUAL or AUTO..

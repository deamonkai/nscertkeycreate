cert_precheck_result

Configuration for API to persist the pre check when the zero touch certificates are uploaded resource.

Read/write properties

id<String>

Id is system generated key for all failed cert entries. .

error_message<String>

Error message if any.

activity_id<String>

Activity Id used to track the progress of all the certificate operations..

file_name<String>

Certificate file name..
Minimum length = 1
Maximum length = 128

file_upload_time<Double>

Creation time for this activity in seconds.

file_data<String>

Encoded cert/key file which has no valid password to decode..
Maximum length = 50000

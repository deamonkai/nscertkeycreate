config_template_temp

Configuration for Configuration Template Temporary file resource.

Read/write properties

description<String>

Description of config template.
Minimum length = 1
Maximum length = 1024

commands<config_command[]>

Array of commands part of the configuration template.

variables<config_variable[]>

Array of variables used in commands of the configuration template.

end_time<Double>

End Time of Interval for which configuration changes are recorded.

commands_str<String>

Configuration Template Commands separated by new line character.

id<String>

Id is system generated key for all the configuration templates.

ip_address<String>

IP Address for which configuration temaplte generated/requested.
Minimum length = 1
Maximum length = 64

template_info<configuration_template>

Configuration Template to be executed/scheduled.

name<String>

Name of config template based on action.
Minimum length = 1
Maximum length = 128

duration<String>

Duration today/last_1_day/last_1_week/current_month/last_1_month of Interval for which configuration changes are recorded.

variable_vals<variable_values[]>

Values of Variables used in commands of the configuration template.

device_family<String>

Family (ns/sdx) of Devices for which config template is defined..
Minimum length = 1
Maximum length = 64

template_file<String>

Template File that is to be imported.
Maximum length = 256

file_name<String>

File Name.
Maximum length = 256

file_location_path<String>

File Location on Client for upload/download.

templates<String[]>

Templates List that are to be imported.

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE

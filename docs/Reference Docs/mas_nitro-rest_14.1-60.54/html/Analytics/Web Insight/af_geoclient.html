af_geoclient

Configuration for af geoclient API resource resource.

Read/write properties

region<String>

Region Name.

bandwidth<Double>

Average Bandwidth..

total_requests<Double>

Count of URL request in selected geo client..

client_ip_address<String>

Client IP Address.
Minimum length = 1
Maximum length = 64

domain_name<String>

Domain Name..
Maximum length = 2000

region_code<String>

Region Code.

network_latency_server_side<Double>

Network latency server side in selected geo client..

country<String>

Country Name.

network_latency_client_side<Double>

Network latency client side in selected geo client..

device_ip_address<String>

NetScaler IP Address.
Minimum length = 1
Maximum length = 64

application_name<String>

Application Name.
Maximum length = 255

__count<Double>

count of geo location record..

app_unit_name<String>

Vserver Name.
Maximum length = 255

total_user<Double>

Count of user i.e. various client IP addresses in the selected region..

country_code<String>

Country Code.

application_response_time<Double>

Application response time..

total_bytes<Double>

Total bytes accounted by this in selected geo client..

city<String>

City Name.

NOTE: In request payload, Mandatory parameters are marked red and bold.

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE

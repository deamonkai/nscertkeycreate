service_graph_geo

Configuration for af geo location API resource resource.

Read/write properties

city<String>

City Name.

ctnsappname<String>

Vserver Name.
Maximum length = 255

application_name<String>

Application Name.
Maximum length = 255

network_latency_server_side<Double>

Network latency server side in selected geo location..

total_requests<Double>

Count of URL request in selected geo location..

ctnsappname_b<String>

Backend LB vserver.
Maximum length = 255

total_bytes<Double>

Total bytes accounted by this in selected geo location..

application_response_time<Double>

Application response time..

device_ip_address<String>

NetScaler IP Address..
Minimum length = 1
Maximum length = 64

country_code<String>

Country Code.

region_code<String>

Region Code.

total_user<Double>

Count of user i.e. various client IP addresses in the selected region..

latitude<Double>

Latitude.

longitude<Double>

Longitude.

network_latency_client_side<Double>

Network latency client side in selected geo location..

bandwidth<Double>

Average Bandwidth..

domain_name<String>

Domain Name..
Maximum length = 2000

region<String>

Region Name.

__count<Double>

count of geo location record..

country<String>

Country Name.
